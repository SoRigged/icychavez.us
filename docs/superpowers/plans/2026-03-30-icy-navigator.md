# Icy Navigator Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deploy icychavez.us as a static dashboard on Cloudflare Pages, then set up Cloudflare Tunnel on Liliana_PC to serve the Chavez cookbook at cookbook.icychavez.us — all with startup-on-boot.

**Architecture:** Static Astro site on Cloudflare Pages (edge-hosted, always on) + Cloudflare Tunnel from Liliana_PC routing cookbook.icychavez.us to a local Next.js app on port 3000. Both the tunnel and cookbook run as Windows Services via `cloudflared service install` and NSSM respectively.

**Tech Stack:** Astro 5, Tailwind CSS 4, Cloudflare Pages, cloudflared, NSSM, Next.js 16 (cookbook)

**Spec:** `docs/superpowers/specs/2026-03-29-icy-navigator-design.md`

### Task 4: Status page

**Files:**
- Create: `src/pages/status.astro`

- [ ] **Step 1: Create status page**

Create `src/pages/status.astro`:

```astro
---
import Layout from "../layouts/Layout.astro";
import { services } from "../data/services";
---

<Layout title="Status — Icy Navigator">
  <header class="pt-12 pb-8 text-center">
    <a href="/" class="text-3xl font-bold tracking-tight text-zinc-100 hover:text-zinc-300 transition-colors">
      Icy Navigator
    </a>
    <p class="mt-2 text-sm text-zinc-500">Service Status</p>
  </header>

  <main class="flex-1 mx-auto w-full max-w-xl px-4 pb-12">
    <div class="space-y-3" id="status-list">
      {services.map((service) => (
        <div
          class="flex items-center justify-between rounded-lg border border-zinc-800 bg-zinc-900/60 px-5 py-4"
          data-url={service.status === "live" ? service.url : ""}
        >
          <div class="flex items-center gap-3">
            <span class="text-lg">{service.icon}</span>
            <span class="text-zinc-200 font-medium">{service.name}</span>
          </div>
          <span class="status-dot h-3 w-3 rounded-full bg-zinc-700" title="Checking..." />
        </div>
      ))}
    </div>
  </main>

  <footer class="py-6 text-center text-sm text-zinc-600">
    Chavez
  </footer>
</Layout>

<script>
  async function checkServices() {
    const items = document.querySelectorAll("#status-list [data-url]");
    for (const item of items) {
      const url = (item as HTMLElement).dataset.url;
      const dot = item.querySelector(".status-dot") as HTMLElement;

      if (!url) {
        dot.classList.remove("bg-zinc-700");
        dot.classList.add("bg-zinc-600");
        dot.title = "Not configured";
        continue;
      }

      try {
        const res = await fetch(url, { mode: "no-cors", signal: AbortSignal.timeout(5000) });
        dot.classList.remove("bg-zinc-700");
        dot.classList.add("bg-emerald-500", "shadow-[0_0_6px_theme(colors.emerald.500)]");
        dot.title = "Online";
      } catch {
        dot.classList.remove("bg-zinc-700");
        dot.classList.add("bg-red-500", "shadow-[0_0_6px_theme(colors.red.500)]");
        dot.title = "Offline";
      }
    }
  }

  checkServices();
  setInterval(checkServices, 30000);
</script>
```

Note: Uses `mode: "no-cors"` with a timeout. This will show green if the server responds at all (even with an opaque response), red if it times out or the host is unreachable. Good enough for a basic health check from the browser.

### Task 9: Install cloudflared as Windows Service

- [ ] **Step 1: Install the tunnel as a service**

Run PowerShell as Administrator:

```powershell
cloudflared service install
```

This installs `cloudflared` as a Windows Service named "Cloudflared agent" that auto-starts at boot. It reads from `%USERPROFILE%\.cloudflared\config.yml` (or the system-level path `C:\Windows\System32\config\systemprofile\.cloudflared\config.yml`).

**Important:** The service runs as Local System, which has a different user profile. You may need to copy the config and credentials to the system profile:

```powershell
mkdir C:\Windows\System32\config\systemprofile\.cloudflared
copy C:\Users\papas\.cloudflared\config.yml C:\Windows\System32\config\systemprofile\.cloudflared\
copy C:\Users\papas\.cloudflared\<tunnel-uuid>.json C:\Windows\System32\config\systemprofile\.cloudflared\
copy C:\Users\papas\.cloudflared\cert.pem C:\Windows\System32\config\systemprofile\.cloudflared\
```

Update the `credentials-file` path in the copied config.yml to match the new location:

```yaml
credentials-file: C:\Windows\System32\config\systemprofile\.cloudflared\<tunnel-uuid>.json
```

- [ ] **Step 2: Configure no-autoupdate**

Edit the service config (or add to the YAML):

```yaml
no-autoupdate: true
```

Add this line to the top of `config.yml` (both copies if you made a system-profile copy).

- [ ] **Step 3: Start and verify the service**

```powershell
net start cloudflared
```

Or via Services UI (services.msc) — find "Cloudflared agent", verify it's Running and set to Automatic startup.

Verify: Visit `https://cookbook.icychavez.us` (with the cookbook also running) — should load.
